Some basic concepts
1. We pull from the main project repo (keep it as in sync as possible)
2. We push into our personal fork of project repo (no need for "collaborator")
3. PR management (ought to be manual) becomes easy on the main project repo

Etiquette on note editing:
1. Don't edit too much at a time for one single note, or there will be edit conflicts between people that gets too hard to handle
2. Ideally we should have top-to-bottom chat/discussion threads in dedicated notes/folders, that way the diff says somewhat consistent without overlapping edit bounds
3. We need citation source storage in dedicated folders to not clutter the space
4. Add links to sources will make searching documents easier
